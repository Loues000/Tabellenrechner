Tabellenrechner

This repository contains a Next.js football table calculator that imports live competition data from fussball.de. The production app now runs on Vercel; this static page only hands off to the live deployment and the source repository.

Production deployment

The real app is live on Vercel.

The application lives in webapp/ and depends on Next.js API routes to fetch, parse, and decode live upstream data. That server-side runtime is now provided by Vercel. This root page stays around only as a lightweight static entry point.

What to do here

Use the live Vercel deployment if you want the actual Tabellenrechner. Use the repository if you want to inspect or extend the codebase.

  1. Open the live app on Vercel.
  2. Browse the repository and README for architecture details.
  3. Run npm run dev from the repository root for local work.
  4. Use webapp as the Vercel root if you create another deployment.

Local run

From the repository root, use npm run dev. The root scripts proxy into webapp.

Repository layout

The application code is in webapp. Samples, task tracking, and this static handoff page stay at the repository root.

Important constraint

The importer fetches live fussball.de data server-side, so a pure static export is not enough without refactoring the architecture.
